# 공지사항 팝업 내 목록 뷰 추가 결과보고서

날짜: 2026-04-21

## 변경 사항 요약
대시보드 알림 바 팝업(`NoticeViewDialog`)에 **"전체 목록"** 버튼을 추가하고, 같은 팝업 안에서 목록/상세 뷰를 토글할 수 있게 했다.
- 별도 페이지(`/notices`) 만드는 대신 팝업 하나로 해결 (사용자 요청에 따라)
- 관리자 `/admin/notices`의 NoticeTable은 영향 없음 (`notices` prop 미전달 시 "전체 목록" 버튼 숨김)

## 동작
1. 대시보드 마키 클릭 → 첫 번째 공지 **상세** 뷰
2. 상세 뷰 푸터의 **"전체 목록"** 버튼 클릭 → 같은 팝업이 **목록** 뷰로 전환
3. 목록에서 항목 클릭 → 해당 공지 상세 뷰로 복귀 (현재 선택 공지는 하이라이트)
4. 상세 뷰의 **"상세로"** 버튼(← arrow)으로 직전 선택 공지 상세로 이동 가능
5. 목록이 비어있으면 빈 상태 UI 표시

## 변경 파일 (2개)

### `components/admin/NoticeViewDialog.tsx`
- `notices?: Notice[]` prop 추가 (있을 때만 목록 모드 활성)
- 내부 `mode: 'detail' | 'list'` 상태 관리
- 팝업 열릴 때마다 상세 모드로 초기화 (`useEffect`)
- 목록 뷰: 카드형 버튼 목록 (제목/내용 미리보기 2줄/작성자/등록일)
- 현재 상세 공지는 주황 포인트로 하이라이트
- 스크롤은 팝업 내부 `max-h-[90vh]`로 자연 처리

### `app/(dashboard)/_components/notice-marquee.tsx`
- `NoticeViewDialog`에 `notices` prop으로 전체 활성 공지 배열 전달
- `author.name`을 `authorName`으로 매핑 (기존 Dialog 인터페이스에 맞춤)

## 설계 결정

### 왜 별도 페이지(`/notices`)가 아니라 팝업 안?
- 공지는 보통 그 자리에서 확인만 하고 넘어가는 흐름 → 페이지 이동이 불필요한 맥락 전환을 유발
- 별도 페이지 만들면 breadcrumb, 메뉴, 레이아웃 고민까지 딸려옴
- 공지 건수가 적어 팝업 내부 스크롤로 충분

### 향후 확장 방향 (건수 많아질 때)
- 최근 50건 이상 쌓이면 "더 보기" 버튼 추가 (초기 N건 + 추가 로드)
- 또는 "최근 N개월" 필터 기본 적용

### 관리자 페이지 호환
- `/admin/notices`의 `NoticeTable`은 `notices` prop을 넘기지 않으므로 "전체 목록" 버튼이 안 보임
- 관리자는 이미 테이블에서 전체 목록을 보고 있어 중복 UI 불필요

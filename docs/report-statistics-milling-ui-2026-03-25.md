# 결과보고서 — 통계 도정실적 페이지 기능·UI 개선

> 작성일: 2026-03-25
> 작업 범위: `statistics/milling` 페이지 필터·그래프·테이블 전반 개선

---

## 변경 사항 요약

### 1. 생산자 검색 추가
- 기간검색 탭 필터 행 우측에 **생산자명 텍스트 입력 + 검색 버튼** 추가
- 쉼표(`,`)로 여러 생산자 동시 검색 (`홍길동, 김철수` → OR 검색)
- Enter 키도 검색 동작
- `statistics.ts` `MillingStatsParams`에 `farmers?: string[]` 추가, Prisma `where.stocks.some.farmer.name IN` 조건 반영

### 2. 검색 버튼 수동 트리거
- 기존: 드롭다운 선택 시 자동 fetch → **검색 버튼 클릭 시에만 fetch**로 변경
- 칩 X 버튼(이미 적용된 조건 제거)은 즉시 fetch (명시적 제거 동작이므로)
- 초기화 버튼: 전체 조건 리셋 후 즉시 fetch

### 3. 검색 조건 칩 행
- 필터 행 아래 **항상 표시**되는 칩 행 추가
- 기간(회색, X 없음) / 품종(파랑) / 도정구분(보라) / 생산자(초록) 색상 구분
- 각 칩 X 클릭 시 해당 조건 제거 + 즉시 fetch

### 4. 그래프 범례 및 Y축
- 범례 투입량(`w-3 h-3`) / 생산량(`w-2 h-3` → `w-3 h-3`) 두께 통일
- 좌측 Y축 `computeTonTicks()` — kg 기준으로 t 단위의 보기 좋은 눈금 계산
- 툴팁도 `0.XX t` 형식으로 통일

### 5. 요약 카드 그라데이션
- 투입량: `from-sky-100/70`
- 생산량: `from-green-100/70`
- 수율: `from-amber-100/70`
- 건수: `from-slate-100/80`

### 6. 상세 테이블 개선
| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 생산자 컬럼 | 전체 이름 쉼표 나열 | "홍길동 외 2명" 요약, hover로 전체 확인 |
| 투입목록 팝업 | 커스텀 모달 | `MillingStockListDialog` 재사용 (`canDelete=false`) |
| 포장내역 팝업 | 플랫 테이블 | AddPackagingDialog 동일 구조 (로트 그룹 헤더) |
| 로트번호 | 텍스트만 | hover 시 생산자명 tooltip 표시 |
| 수율 표시 | 색상 분기 `toFixed(1)%` | 70%↑ 파란 배지, 미만 회색 배지 |

## 주요 결정 사항

- **수동 검색 방식 채택**: 드롭다운 변경 → 자동 fetch는 UX 노이즈(변경 중에 계속 서버 요청)가 크다는 판단. 검색 버튼 명시 트리거로 변경.
- **칩 X 제거는 즉시 fetch**: "이미 적용된 조건을 제거"는 명시적 의사 표현이므로 즉시 반영.
- **MillingStockListDialog 재사용**: 도정관리와 완전 동일한 투입목록 화면 제공. `StockDetail`에 `id`, `varietyType`, `certType` 추가 필요.
- **PackagingPopup 재설계**: `AddPackagingDialog`는 편집 로직이 복잡하게 얽혀 있어 직접 재사용 불가. 동일 레이아웃(로트 그룹 헤더)을 읽기전용으로 별도 구현.

## 확인이 필요한 사항

- [ ] Y축 t 단위 — 소량 데이터(100kg 이하)에서 `0.0t` 레이블이 어색할 수 있음. 실제 데이터로 확인 필요.
- [ ] 포장내역 팝업 로트 그룹핑 — `outputDetail.stockId` 기반으로 그룹핑하므로, 관행 농가 여러 명이 같은 stockId를 가질 경우 합산될 수 있음. 실제 운용 중 데이터로 검증 필요.
- [ ] 생산자 쉼표 검색 — `farmer.name IN [...]` 완전 일치 방식. 부분 일치가 필요하면 추후 `contains` 방식으로 변경 고려.
